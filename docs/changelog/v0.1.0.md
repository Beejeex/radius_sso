# v0.1.0 Detailed Changelog

Milestone: Identity, Access & UI Shell
Status: Complete
Summary: [CHANGELOG.md](../../CHANGELOG.md)

## Added

- Added first-run setup mode for initial OIDC provider configuration without a setup key.
- Added native ASP.NET Core browser authentication using named `AddOpenIdConnect()` schemes and `AddCookie("WebCookie")`.
- Added the dedicated `JellyfinTokenHandler` for Jellyfin-compatible `MediaBrowser` header authentication.
- Added `/api/auth/me` and web user rehydration so the React app can show the signed-in user after page refresh.
- Added the Framework Defaults First agent guidance requiring native framework implementations before custom code.

## Changed

- Changed the active auth architecture to two normal mechanisms only: browser OIDC/cookie auth and Jellyfin-compatible header token auth.
- Changed setup provider creation to require a restart before browser sign-in uses the configured native `Oidc` scheme.
- Changed active ADRs, specs, schema docs, project state, and Acceptance tracking to reflect the current framework-first auth architecture.
- Changed local development validation to allow HTTP OIDC authorities for loopback, `.local`, and single-label Docker hostnames while keeping production HTTPS validation strict.

## Fixed

- Fixed local setup/OIDC smoke issues found during browser validation on 2026-06-09, including local HTTP authority handling, OIDC callback/sign-in behavior, and current-user display.
- Fixed stale documentation that could lead agents toward obsolete auth flows or custom framework replacements.
- Fixed Dockerfile `HEALTHCHECK` by switching from `wget` to `curl` so the production image can report container liveness without additional shell utilities.
- Fixed mock-oidc container healthcheck to use `GET` instead of `HEAD` for compatibility with the mock OIDC server.
- Fixed S-7 smoke-test false positive (reported `401` as a failure when it is correct for an unauthenticated setup endpoint probe during redirect).

## Removed

- Removed obsolete auth ADRs, outdated test tracking documents, scratch logs, and temporary test-output files that no longer describe the active system.
- Removed historical guidance for browser password sign-in and hand-written browser OIDC flow ownership from active project documents.
