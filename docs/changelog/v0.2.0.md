# v0.2.0 Detailed Changelog

Milestone: Media Library Core & Browse UI
Status: Complete — accepted 2026-06-13
Summary: [CHANGELOG.md](../../CHANGELOG.md)

## Added

- Added library CRUD with Windows path support: domain types (`LibraryDefinition`, `LibraryType`, path validation), application service (`ILibraryManagementService`), EF Core persistence mappings, audit logging for destructive actions, and integration endpoints (`GET/POST/DELETE /Library/VirtualFolders`).
- Added media records for all five library types — `Movie`, `Series`, `Season`, `Episode`, `PersonalMediaItem` — with stable GUID derivation (`MediaItemGuid`), TPT persistence mappings with FK/unique/check constraints, and application query service (`IMediaItemQueryService`).
- Added media library scanning: `IFileSystemEnumerator` (filesystem discovery), `ITvStructureRecognizer` (S01E01/S1E1/1x01 pattern parsing), `IMediaRecordSynchronizer` (idempotent record sync with file-gone/file-restore detection), and `ILibraryScanOrchestrator` (scan trigger, concurrency prevention, status queries) — executed through the background job queue.
- Added access control and user management: `ILibraryAccessService` (grant/revoke/list), `ILibraryAccessFilter` (IQueryable access filtering for per-library grants), `IUserManagementService` (CRUD, disable), `IUserAdminRoleService` (grant/revoke with self-revoke prevention), `IDeletePermissionService`, and domain invariants (`LibraryAccessGrant`, `UserAccount.EnableAllFolders`).
- Added Web UI pages for library home, library browse with TV drill-down (Series → Seasons → Episodes), admin user management with inline editing, admin library management, admin OIDC provider management (system-managed providers greyed out), and authenticated route protection.
- Added audit event system with IP hashing for library and user management actions.
- Added health/readiness endpoints (`GET /health`, `GET /health/ready`) reachable without authentication.
- Added Docker multi-stage production build running as non-root user (final image 365 MB).

## Changed

- Changed user delete FK constraint from `Restrict` to `SetNull` so deleting a user does not block on remaining library access grants or audit events.
- Changed frontend API response handling to apply PascalCase-to-camelCase transformation for consistent consumption in the React client.
- Changed background scan scope and cancellation handling to prevent concurrent scans and support graceful cancellation.

## Fixed

- Fixed audit event type check constraint missing `user_created`, `user_deleted`, and `user_disabled` event types.
- Fixed background scan concurrency and scope management to avoid stale scan state on cancellation.
- Fixed frontend camelCase transformation to correctly map PascalCase API responses for library and media item data.
- Fixed 3 case-sensitivity regressions in GUID derivation and file extension matching.
