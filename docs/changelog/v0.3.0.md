# v0.3.0 Detailed Changelog

> Metadata, Artwork & Item Detail UI — **676/676 tests GREEN** (Domain 268, Application 367, Persistence 41)
> 22 stories delivered across 5 increments.

## [0.3.1] — Metadata Foundation & Movie TMDB Provider

### Added
- Defined `IMetadataProvider` abstraction for external metadata source integration.
- Implemented TMDB movie metadata provider: plot synopsis, cast, ratings, genres, release year, runtime.
- Built filename-to-query parser extracting year and title for media lookup.
- Added failure handling with retry and graceful degradation when metadata sources are unreachable.
- Wired metadata provider discovery through dependency injection.

## [0.3.2] — TV Metadata & Subtitle Discovery

### Added
- Extended metadata providers for TV series, season, and episode content via TMDB.
- Built TV filename parser recognizing `SeriesName S01E02` and similar patterns.
- Added sidecar subtitle file discovery for `.srt`, `.vtt`, `.ass`, and `.ssa` formats adjacent to media files.
- Implemented subtitle serving endpoint for Jellyfin-compatible client consumption.

### Changed
- Refactored filename parsing into reusable components shared by movie and TV providers.

## [0.3.3] — Artwork Providers & Artwork API

### Added
- Defined `IArtworkProvider` abstraction for artwork/image source integration.
- Implemented local artwork discovery scanning for poster, backdrop, and logo files alongside media.
- Implemented TMDB artwork provider for poster and backdrop images with community API key default.
- Built artwork store service managing artwork metadata and storage paths.
- Added `GET /Items/{id}/Images/{kind}` compatibility endpoint serving artwork by type.

### Fixed
- Artwork file path validation to prevent traversal outside library roots.

## [0.3.4] — Item Detail Endpoint & Metadata Refresh

### Added
- Added `GET /api/items/{id}` detail endpoint returning full metadata, artwork references, cast, and subtitle availability.
- Implemented admin-triggered metadata refresh via Hangfire background job queue.
- Added `POST /api/items/{id}/refresh` endpoint for per-item metadata re-fetch.
- Wired Hangfire job orchestration: scan-triggered metadata enrichment on library scan completion.
- Added structured logging for metadata refresh outcomes and failures.

## [0.3.5] — React Item Detail Page

### Added
- Built rich item detail page at `/items/{id}` with responsive hero section.
- Displayed backdrop artwork as page header with poster overlay.
- Rendered full metadata panel: title, year, runtime, rating, genres, synopsis.
- Added cast list section with character names.
- Displayed available subtitle tracks with download links.
- Added playback stub button for future playback integration.
- Implemented loading state skeleton and error state for failed data fetches.
- Applied responsive layout supporting mobile, tablet, and desktop viewports.

## [0.3.*] — Cross-Increment

### Added
- Full test coverage for all metadata providers, artwork providers, filename parsers, and subtitle services.
- TMDB community API key configured as default for out-of-box metadata fetching.
- Integrated metadata refresh as post-scan Hangfire job for automatic enrichment.

### Changed
- Updated library scan pipeline to trigger metadata enrichment as a background job.
- Persisted artwork references alongside media records for efficient query.

### Fixed
- Registered missing `IItemLookup` service in DI to resolve activation errors during metadata refresh and item detail lookup; created `ItemLookup.cs` implementation and wired it in `ApplicationServiceCollectionExtensions.cs`.

### Security
- Validated all external API responses against expected schemas to prevent injection.
- Sanitized user-provided search terms passed to external metadata APIs.
