# Greenfield Project Template

This repository is a neutral starting point for a new software project. It keeps the project workflow, agent roles, decision records, specifications, schemas, testing templates, and state tracking in one place without assuming a product domain or technology stack.

Start with [STARTING_GUIDE.md](STARTING_GUIDE.md), then fill in [Project State](docs/project/STATE.md) before adding implementation code.

## Repository Areas

- `.github/` - reusable agent flow, instructions, skills, and handoff prompts.
- `docs/project/` - current state, roadmap, backlog, and project-wide guidance.
- `docs/specs/` - product and behavior specifications.
- `docs/adr/` - architecture and technology decisions.
- `docs/schema/` - durable data design.
- `docs/testing/` - test and validation plans.
- `docs/changelog/` - delivered changes.
